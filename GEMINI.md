## Gemini Operational Guidelines for Git

As Gemini, when working on this project, always adhere to the following Git workflow:

1.  **Pull:** Before making any changes, always pull the latest changes from the remote repository to ensure you are working on the most up-to-date codebase.
2.  **Edit:** Proceed with the requested code modifications or file operations.
3.  **Test:** Run Playwright tests (`npm run test-e2e`) and fix any issues.
4.  **Commit:** After completing a logical unit of work, commit your changes with a clear and concise commit message.
5.  **Push:** After committing, push your changes to the remote repository.
